# 🌐 FormFlow Pro Production Deployment Guide

This documentation guides you through configuring, managing, and maintaining **FormFlow Pro** in a production-ready Node.js environment.

---

## 🛠️ Architecture Overview

FormFlow Pro is a full-stack, state-of-the-art form building and survey application built with:
- **Backend**: Node.js Express server (`server.ts`), compiled into a self-contained CJS bundle (`dist/server.cjs`) via `esbuild`.
- **Frontend**: React SPA styled with Tailwind CSS, built into static assets served from the backend (`dist/`).
- **Database**: Standard JSON file database with dynamic directory configurations and automated rotating backups.

---

## 🔑 Environment Settings (`.env`)

To configure the application for production, create a `.env` file in your root directory based on the `.env.example` file:

```ini
# Host Settings
PORT=3000
NODE_ENV=production

# Security & Secrets
JWT_SECRET=your_super_stable_jwt_secret_key_change_me_in_production
CORS_ORIGIN=*

# Rate Limiting
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX=1000

# File Paths (Relative or Absolute paths outside of Docker container or VPS scope)
DB_FILE_PATH=./db.json
UPLOADS_DIR_PATH=./uploads
EXCEL_DIR_PATH=./excel_storage
BACKUP_DIR_PATH=./backups

# Automated Backups
ENABLE_BACKUPS=true
BACKUP_INTERVAL_MS=86400000
```

## 🚀 Deployment Options

Choose the strategy that best fits your infrastructure:

### Option A: Using Docker & Docker Compose (Recommended)
This is the most secure, reproducible, and self-contained approach.

1. **Verify Docker and Docker Compose are installed on your server**:
   ```bash
   docker --version
   docker compose version
   ```

2. **Configure your environment**:
   Make sure to create/edit your `.env` file on the server.

3. **Start the containers**:
   ```bash
   docker compose up -d --build
   ```
   *This commands builds the image in stages, sets up persistent volumes under `formflow_prod_data`, and launches the application container on port `3000`.*

4. **Verify container status**:
   ```bash
   docker compose ps
   docker compose logs -f
   ```

---

### Option B: PM2 Process Management (Traditional VPS/Ubuntu)
Use PM2 to run the application in Cluster mode across multiple CPU cores, with zero-downtime reloads.

1. **Install Node.js 20+ and PM2 globally on your server**:
   ```bash
   # Install Node.js
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt-get install -y nodejs

   # Install PM2
   sudo npm install pm2 -g
   ```

2. **Clone and build your repository**:
   ```bash
   git clone <repo-url> /var/www/formflow-pro
   cd /var/www/formflow-pro
   npm install
   npm run build
   ```

3. **Start PM2 in cluster mode**:
   ```bash
   pm2 start ecosystem.config.cjs
   ```

4. **Persist PM2 across server restarts**:
   ```bash
   pm2 startup
   # Copy and execute the command generated by the startup script
   pm2 save
   ```

5. **Monitor performance and logs**:
   ```bash
   pm2 status
   pm2 monit
   pm2 logs formflow-pro
   ```

---

### Option C: PaaS Hosting Providers (Render, Heroku, Railway)
If deploying to modern fully-managed container platforms:

1. **Build command**: `npm install && npm run build`
2. **Start command**: `npm start` (which runs `node dist/server.cjs`)
3. **Environment variables**: Register `PORT`, `NODE_ENV=production`, `JWT_SECRET`, etc., in the platform's Environment/Secrets configuration panel.

## 🗄️ Persistence and Storage Strategy

FormFlow Pro does not require a bulky external SQL setup, operating instead with zero-configuration efficiency. However, in production, ensuring files are written to non-volatile volumes is crucial:
- **`uploads/`**: Stores uploaded files, signatures, and pictures.
- **`excel_storage/`**: Stores compiled XLS/CSV exports.
- **`backups/`**: Stores automatic copies of `db.json` (rotated to keep the last 10 backups).

### VPS Directory Permissions
Ensure that the user running the Node.js application has write permissions for those directories:
```bash
mkdir -p uploads excel_storage backups
chmod -R 755 uploads excel_storage backups
```
In Docker-based deployments, this is automated via the `Dockerfile` with standard container volume binds.
